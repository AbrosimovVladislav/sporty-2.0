# Роадмап реализации MVP

## Итерация 0: Инфраструктура

**Цель:** рабочая среда — Supabase проект, Telegram-бот, деплой на Vercel.

- ✅ **0.1** Создать проект в Supabase, получить URL + ключи
- ✅ **0.2** Создать Telegram-бота через BotFather, получить токен
- ✅ **0.3** Первый деплой на Vercel, получить URL
- ✅ **0.4** Настроить Mini App в BotFather (Menu Button → URL Vercel)
- ⬜ **0.5** Заполнить `.env.local` и Vercel env vars

**Зависимости:** нет

---

## Итерация 1: Фундамент

**Цель:** пользователь открывает Mini App, авторизуется, проходит онбординг.

- ✅ **1.1** Supabase — установить `@supabase/supabase-js`, создать `src/lib/supabase.ts`
- ✅ **1.2** Применить миграцию: таблица `User`
- ✅ **1.3** Auth API — `POST /api/auth/telegram`: валидация initData, upsert user
- ✅ **1.4** Client auth — хук `useAuth` + `AuthProvider`
- ✅ **1.5** Онбординг UI — выбор роли → форма игрока → главная-заглушка

**Зависимости:** Итерация 0

---

## Итерация 2: Команда

**Цель:** организатор может создать команду и управлять составом.

- ⬜ **2.1** Миграции: таблицы `Team`, `TeamMembership`
- ⬜ **2.2** Онбординг организатора — создание команды (название, спорт, город)
- ⬜ **2.3** API: CRUD команды, управление составом
- ⬜ **2.4** Экраны: главная команды, профиль команды, список игроков

**Зависимости:** Итерация 1

---

## Итерация 3: Заявки и поиск

**Цель:** игрок находит команду и подаёт заявку на вступление.

- ⬜ **3.1** Миграция: таблица `JoinRequest`
- ⬜ **3.2** Экран поиска команд (фильтр по городу и спорту)
- ⬜ **3.3** Отправка заявки, экран «Мои заявки» (игрок)
- ⬜ **3.4** Входящие заявки (организатор): принять / отклонить
- ⬜ **3.5** API: поиск, создание и обработка заявок

**Зависимости:** Итерация 2

---

## Итерация 4: События

**Цель:** организатор создаёт событие, игроки голосуют.

- ⬜ **4.1** Миграции: таблицы `Venue`, `Event`, `EventAttendance`
- ⬜ **4.2** Создание события (тип, дата, площадка, цена, мин. игроков)
- ⬜ **4.3** Карточка события, список игроков с голосами
- ⬜ **4.4** Голосование: приду / не приду
- ⬜ **4.5** Лента событий на главной игрока и команды
- ⬜ **4.6** API: CRUD событий, голосование

**Зависимости:** Итерация 2

---

## Итерация 5: Результаты и финансы

**Цель:** после события — учёт посещений и денег.

- ⬜ **5.1** Экран отметки игрока: был / сдал деньги
- ⬜ **5.2** Подтверждение организатора: корректировка отметок
- ⬜ **5.3** Финансовая сводка по событию: ожидаемый сбор, фактический, разница
- ⬜ **5.4** История событий игрока, статистика на главной команды

**Зависимости:** Итерация 4

---

## Порядок и параллелизм

```
Итерация 1 (фундамент)
    ↓
Итерация 2 (команда)
    ↓
   ┌──────────────┐
   ↓              ↓
Итерация 3     Итерация 4
(заявки)       (события)
               ↓
            Итерация 5
            (финансы)
```

Итерации 3 и 4 можно вести параллельно — они независимы друг от друга.
