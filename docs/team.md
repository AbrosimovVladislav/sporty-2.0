# Команда

## Модель ролей

Роль пользователя — это свойство его связи с командой (`TeamMembership.role`), а не свойство самого пользователя. Один человек может быть игроком в одной команде и организатором в другой одновременно.

### Роли внутри команды

- **player** — участник команды, голосует за события, видит общий состав и расписание.
- **organizer** — участник команды с расширенными правами: управляет составом, создаёт события, видит финансы команды, принимает заявки на вступление.

В одной команде может быть **несколько организаторов**. У всех одинаковые права. Первый организатор команды — это её создатель.

### Как стать организатором

1. **Создать команду** — пользователь становится первым организатором новой команды.
2. **Получить промоут** от существующего организатора — игрок команды повышается до организатора. Существующий организатор меняет `TeamMembership.role` с `player` на `organizer`.

Заявка «стать организатором извне» (не будучи игроком команды) — отложена за MVP, см. [post-mvp.md](post-mvp.md).

## Создание команды

Доступно любому пользователю. Точка входа: экран «Команды», secondary-ссылка «Создать команду».

**Форма создания:**
- Название команды
- Город
- Вид спорта (на старте — только футбол)

При создании:
1. Создаётся запись в `Team`.
2. Создаётся запись в `TeamMembership` с `user_id = текущий пользователь`, `role = 'organizer'`.
3. Пользователь попадает на главную команды (`/team/[id]`).

## Экран команды (`/team/[id]`)

Единый роут с разным содержимым в зависимости от отношения пользователя к команде.

### Три режима просмотра

1. **Не-участник** — пользователь попал на команду из каталога (`/teams` → «Все команды»). Видит публичный профиль команды (только основная инфа) и кнопку «Подать заявку».
2. **Игрок команды** — полный доступ к жизни команды, но без финансов и управленческих действий.
3. **Организатор команды** — всё то же, что видит игрок, плюс финансовый блок и управленческие действия.

Режим определяется запросом к `TeamMembership`: есть ли у текущего `user_id` запись с этим `team_id` и какая роль.

---

## Публичный вид (не-участник)

Ограниченный view для показа команды «снаружи».

**Содержимое:**
- Название, город, спорт
- Описание (если задано)
- Количество игроков в составе
- Кнопка **«Подать заявку на вступление»** (если заявки ещё нет)
- Либо надпись «Заявка отправлена» / «Заявка отклонена» (если уже подавал)

Вложенные вкладки (Главная / Состав / События) в этом режиме **не показываются**. Только карточка команды и действие.

## Вид участника (player и organizer)

Когда пользователь уже состоит в команде, открывается полноценный экран команды с верхней суб-навигацией.

### Суб-табы команды

```
Главная    |    Состав    |    События
```

Одинаковые три вкладки для игроков и организаторов. Различия — внутри экранов: часть блоков и кнопок видна только организаторам.

**Важно:** это суб-навигация *внутри* экрана команды. Глобальные табы снизу (Главная / Команды / Профиль) по-прежнему видны — пользователь может в любой момент уйти в свой профиль или к списку команд.

---

## Таб «Главная команды»

Дашборд команды. Первое, что видит пользователь после захода в команду.

**Видит любой участник:**
- Заголовок: название команды, город, спорт
- Количество игроков в составе
- Ближайшее событие (карточка)

**Дополнительно видит организатор:**
- Блок **«Финансовый баланс команды»**: сколько собрано, сколько потрачено, остаток
- Блок **«Входящие заявки (N)»** с переходом на список заявок (если N > 0)
- Кнопка **«Настройки команды»** (редактирование названия, города, описания)

---

## Таб «Состав»

Список участников команды.

**Видит любой участник:**
- Карточка игрока: имя, дата вступления в команду, бейдж роли (organizer / player)
- Список отсортирован: сначала организаторы, потом игроки

**Дополнительно видит организатор:**
- Кнопка **«Удалить»** рядом с каждым игроком (не применима к самому себе, если он единственный организатор)
- Кнопка **«Сделать организатором»** рядом с игроком (промоут)

---

## Таб «События»

Список событий команды — предстоящих и прошедших.

**Видит любой участник:**
- Список карточек событий: тип, дата, площадка, статус голосования
- Кнопка голосования «приду / не приду» прямо в карточке
- Клик по карточке → детальный экран события (см. [event.md](event.md))

**Дополнительно видит организатор:**
- Кнопка **«Создать событие»** сверху экрана
- Внутри карточки события — краткий финансовый индикатор (например, «сбор 4500 из 6000 ₽»)

Игрок финансовые индикаторы не видит — только информацию о том, сколько нужно сдать за конкретное событие, в котором он участвует.

## Заявки на вступление

Точка входа для организатора: блок **«Входящие заявки (N)»** на главной команды. Клик → отдельный список входящих заявок.

### Список входящих заявок (только организатор)

- Карточка заявки: имя игрока, город, дата заявки
- Действия: **принять** / **отклонить**

### Флоу заявки (со стороны игрока)

1. Игрок находит команду в каталоге (`/teams` → «Все команды»).
2. Открывает публичный профиль команды.
3. Нажимает «Подать заявку на вступление» → создаётся `JoinRequest` со статусом `pending`.
4. В своём профиле (`/profile` → «Мои заявки») игрок видит статус.
5. Любой организатор команды принимает или отклоняет заявку:
   - **Принято** → `JoinRequest.status = 'accepted'`, создаётся `TeamMembership` с `role = 'player'`. Команда появляется в «Мои команды».
   - **Отклонено** → `JoinRequest.status = 'rejected'`, в списке заявок у игрока остаётся запись со статусом.
