# Пользователь и навигация

Единая модель: в приложении нет разделения на «игрока» и «организатора» на уровне пользователя. Есть просто пользователь с профилем. Роли появляются внутри конкретных команд (см. [team.md](team.md)).

Соответственно и главная, и навигация, и профиль — одинаковые для всех.

---

## Глобальная навигация

Снизу фиксируются три таба, **всегда видимые** на всех экранах внутри приложения (в том числе внутри экрана команды):

```
[ Главная ]    [ Команды ]    [ Профиль ]
```

| Таб | Роут | Что содержит |
|-----|------|--------------|
| Главная | `/home` | Лента ближайших событий из всех команд пользователя |
| Команды | `/teams` | Мои команды + каталог всех команд с поиском |
| Профиль | `/profile` | Личная информация и настройки |

Глобальные табы не исчезают при переходе в детальные экраны (экран команды, экран события, профиль команды). Это обеспечивает, что профиль и навигация между основными разделами всегда под рукой.

---

## Главная (`/home`)

Лента предстоящих событий из всех команд, где пользователь состоит.

**Карточка события в ленте:**
- Тип события (игра / тренировка / сбор)
- Дата и время
- Название команды (с подписью «ты в составе»)
- Площадка
- Быстрое голосование «приду / не приду» прямо из карточки

**Empty state** (пользователь не состоит ни в одной команде):
- Большой блок с сообщением «Ты пока не в команде»
- Кнопка **«Найти команду»** → переводит на таб «Команды», секция «Все команды»

---

## Команды (`/teams`)

Экран с двумя под-секциями (переключаются сверху pill-табами):

### 1. Мои команды

Список команд, в которых пользователь состоит.

**Карточка команды:**
- Название, город, спорт
- Количество игроков в составе
- Бейдж роли: «организатор» или «игрок»

Клик по карточке → `/team/[id]`.

**Empty state:** «Ты пока не в команде» + кнопка «Посмотреть все команды» (переключает на соседнюю под-секцию).

Внизу экрана — secondary-ссылка **«Создать команду»** (не primary, чтобы не перегружать новичков). Ведёт на форму создания команды (см. [team.md](team.md)).

### 2. Все команды (каталог)

Каталог команд для поиска и вступления.

**Фильтры сверху:**
- Город
- Вид спорта (на старте — только футбол, поле зафиксировано)

**Результат — карточки команд:**
- Название, город, спорт
- Количество игроков

Клик по карточке → публичный профиль команды (`/team/[id]`, но в режиме «не-участника», см. [team.md](team.md)). Оттуда можно подать заявку на вступление.

---

## Профиль (`/profile`)

Личные данные пользователя и его активность.

**Содержимое:**
- Имя, город, вид спорта
- **События** — история событий пользователя (последние 10): тип, команда, дата, бейджи (приду/не приду/был). Сверху — сводка: завершённых событий, посещено из N
- **Мои заявки** — список поданных заявок на вступление в команды со статусами (`на рассмотрении` / `принята` / `отклонена`)
- Настройки (пока минимум, расширение в пост-MVP)

Профиль — это «я как пользователь», без привязки к конкретной команде. Отдельного «профиля игрока» и «профиля организатора» нет — на уровне пользователя есть один профиль.

---

## Переходы между экранами

```
/home → карточка события → /team/[id]/events/[eventId]
/home (empty) → «Найти команду» → /teams (секция «Все команды»)

/teams «Мои» → карточка команды → /team/[id]
/teams «Все» → карточка команды → /team/[id] (вид не-участника)
/teams «Мои» → «Создать команду» → форма создания → /team/[id]

/profile → «Мои заявки» → список заявок
```

Глобальные табы остаются внизу в любом из этих экранов. Возврат — через системный back или через табы.
