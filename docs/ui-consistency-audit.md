Карта расхождений между текущим интерфейсом и [дизайн-системой](design/design-system.md). Каждая запись — отдельная задача для итераций 24–29 (см. [mvp-roadmap.md](mvp-roadmap.md)).

Перед любой UI-задачей:
1. Прочитать [design/design-system.md](design/design-system.md) целиком.
2. Найти соответствующий пункт ниже.
3. После реализации — отметить пункт как «✅ закрыт в итерации N».

## 1. Глобальные токены и фундамент

### 1.1 Олива вместо vivid green
- **Проблема.** В [src/app/globals.css](../src/app/globals.css) `--primary: #4d5e23` (приглушённая олива). На референсе primary — vivid green `#22C55E`.
- **Почему плохо.** Бренд выглядит «защитного» цвета, не спортивно. Все «приду», «оплачено», табы — тусклые.
- **Как привести.** Обновить токены `--primary`, `--primary-hover`, добавить `--primary-soft`. См. таблицу токенов в дизайн-системе.
- **Файлы.** [src/app/globals.css](../src/app/globals.css). Все экраны проверить визуально — изменения подхватятся через переменную.

### 1.2 Отсутствуют семантические токены warning/danger/success
- **Проблема.** В коде везде `text-green-600`, `text-red-500`, `bg-red-500/10` напрямую из Tailwind. Цвета не централизованы.
- **Почему плохо.** При смене палитры нужно менять в 30+ местах. Невозможно гарантировать единый красный/оранжевый.
- **Как привести.** Добавить токены `--danger`, `--warning`, `--success` (alias `--primary`) и их `-soft` версии в [globals.css](../src/app/globals.css), пробросить через `@theme inline`. Заменить хардкод на токены.
- **Файлы.** [src/app/globals.css](../src/app/globals.css), затем массовый поиск/замена `text-green-600`/`bg-green-600`/`text-red-500`/`bg-red-500` во всех `src/app/**/*.tsx`, `src/components/**/*.tsx`.

### 1.3 Нет токенов для теней
- **Проблема.** Карточки используют `border border-border`. Теней нет.
- **Почему плохо.** Граница даёт «жёсткий» вид; тень — мягкий iOS-стиль референса.
- **Как привести.** Добавить `--shadow-card`, `--shadow-pop`. Обновить дефолтную карточку: убрать border, добавить shadow.
- **Файлы.** [globals.css](../src/app/globals.css), все `bg-background-card border border-border` (массово).

### 1.4 Шрифт Oswald пронизывает интерфейс
- **Проблема.** В [design/design-system.md](design/design-system.md) Oswald был объявлен дисплейным. Используется в 100+ местах: эйбрау-метках секций, кнопках, бейджах, hero-заголовках.
- **Почему плохо.** Condensed uppercase читается тяжелее, выглядит «спортивная газета 2010-х», не iOS.
- **Как привести.** Снять `font-display` и `uppercase` со всех заголовков карточек, бейджей, эйбрау. Оставить только `font-semibold`/`font-bold`. Эйбрау-метки секций — `text-[11px] uppercase tracking-wide font-semibold` (uppercase сохраняется, но шрифт обычный sans).
- **Файлы.** массово. Search: `font-display`. Большинство удаляем; держим Oswald только для редких маркетинг-сценариев (логотип, баннеры).

### 1.5 Тёмные hero-блоки на не-фото-экранах
- **Проблема.** [team/[id]/layout.tsx](../src/app/(app)/team/[id]/layout.tsx#L57), [home/page.tsx](../src/app/(app)/home/page.tsx#L88) — большие тёмные `bg-background-dark` плашки с именем/командой.
- **Почему плохо.** На референсе тёмный фон оправдан только под фото с градиентом. Иначе он выглядит тяжело и устаревше.
- **Как привести.** Заменить hero на «светлый хедер»: `<` назад + `text-[28px] font-bold` заголовок + IconButton-ы справа (см. дизайн-систему «Хедер экрана»).
- **Файлы.** [src/app/(app)/team/[id]/layout.tsx](../src/app/(app)/team/[id]/layout.tsx), [src/app/(app)/home/page.tsx](../src/app/(app)/home/page.tsx), [src/app/(app)/profile/page.tsx](../src/app/(app)/profile/page.tsx).

## 2. Главная страница (`/home`)

### 2.1 Тёмное приветствие
- **Проблема.** Welcome header — тёмный блок с Oswald-uppercase именем (см. 1.5).
- **Как привести.** «Привет, Иван» как `text-[28px] font-bold` на светлом фоне; под ним `text-foreground-secondary` город/спорт.

### 2.2 Карточка ближайшего события дублирует тёмный стиль
- **Проблема.** [home/page.tsx:108–134](../src/app/(app)/home/page.tsx#L108) — `bg-background-dark` карточка ближайшего события без фото.
- **Почему плохо.** Не совпадает с фото-баннером события на детальном экране — разный визуальный язык в одном flow.
- **Как привести.** Использовать карточку с фото-баннером из дизайн-системы. Если у площадки нет фото — градиентный плейсхолдер. Добавить триплет-метрику и кнопки голосования (приду/не приду) прямо на главной для самого ближайшего.
- **Файлы.** [home/page.tsx](../src/app/(app)/home/page.tsx). Возможно понадобится поле `venues.photo_url` (отдельная подзадача — в роадмапе).

### 2.3 «Найти игру / Команды» — серые невыразительные плашки
- **Проблема.** [home/page.tsx:138–153](../src/app/(app)/home/page.tsx#L138) — две одинаковые `bg-background-card border border-border`.
- **Как привести.** Использовать carbon-кнопки с большой иконкой сверху и подписью. Один primary CTA если у юзера нет команд (главная задача — найти игру), второй secondary.

### 2.4 «Мои команды» без статистики
- **Проблема.** Карточка команды показывает только имя, город, спорт, роль.
- **Почему плохо.** На референсе строки сильнее насыщены данными (мини-бар, число).
- **Как привести.** Добавить рядом с ролью индикатор активности (например, ближайшее событие через N дней или «нет событий») — подпись `text-[13px] text-foreground-secondary`.

## 3. События — лента (`/team/[id]/events`)

### 3.1 Нет фото-обложек
- **Проблема.** [events/page.tsx:134–183](../src/app/(app)/team/[id]/events/page.tsx#L134) — карточка события только текст.
- **Как привести.** Добавить мини-обложку 80×80 слева (фото площадки или градиентный плейсхолдер). Тип события — пилюлей поверх.
- **Файлы.** [team/[id]/events/page.tsx](../src/app/(app)/team/[id]/events/page.tsx).

### 3.2 Информация дублируется
- **Проблема.** В одной карточке: «✓ N ✗ N мин. M», + «Ожидаемый сбор», + «Площадка X ₸ (оплачено Y)», + «Вы: приду». Слишком много слоёв.
- **Как привести.** Оставить: дата, площадка, статус (пилюля), счётчик «Придут N / нужно M», ваш статус (мини-чип). Финансы — только в детальном экране.

### 3.3 Кнопка «Создать событие» как primary без иконки
- **Проблема.** [events/page.tsx:80–86](../src/app/(app)/team/[id]/events/page.tsx#L80) — `<button>` с лейблом «Создать событие».
- **Как привести.** Перенести в IconButton `+` в правом углу хедера; primary-кнопка снизу зарезервирована под основное действие на screen.

### 3.4 Форма создания события — длинная плоская
- **Проблема.** Все поля идут сплошной портянкой в одной карточке. Нет шагов/секций.
- **Почему плохо.** На мобильном тяжело сканировать.
- **Как привести.** Сгруппировать в логические блоки-карточки: «Когда» (тип, дата, время), «Где» (площадка), «Сколько» (цена, мин. игроков), «Описание». Каждый — `bg-background-card rounded-lg p-4 shadow-card`.

## 4. Событие — детали

### 4.1 Нет фото-баннера
- **Проблема.** [team/[id]/events/[eventId]/page.tsx:98–133](../src/app/(app)/team/[id]/events/[eventId]/page.tsx#L98) — простая карточка.
- **Как привести.** См. дизайн-систему «Фото-баннер». Если нет фото — градиент-плейсхолдер.

### 4.2 Тип и статус — мелкие пилюли в одной строке, конкурируют с заголовком
- **Проблема.** [event/[eventId]/page.tsx:99–106](../src/app/(app)/team/[id]/events/[eventId]/page.tsx#L99) — обе пилюли в `text-xs`, олива и серый.
- **Как привести.** Поверх баннера — две читаемые статус-пилюли (МАТЧ зелёная + ЗАПЛАНИРОВАНО светлая полупрозрачная).

### 4.3 Триплет-метрика отсутствует
- **Проблема.** Сейчас «Цена» и «Мин. игроков» — мелкие колонки внутри той же карточки. Нет «X / Y участников» с прогрессом.
- **Как привести.** Отдельная карточка под баннером, 3 колонки: взнос, мин. игроков, X / Y участников + мини-прогресс-бар.

### 4.4 «Голоса (N)» дублирует «Придут / Не придут»
- **Проблема.** [event/[eventId]/page.tsx:614–652](../src/app/(app)/team/[id]/events/[eventId]/page.tsx#L614) — общий счётчик «Голоса (N)» сверху, потом «Придут (N)», «Не придут (N)».
- **Как привести.** Убрать «Голоса (N)»; оставить две группы с эйбрау. Аватарки + «Показать всех», без числа справа от стека.

### 4.5 Аватарок нет вообще
- **Проблема.** Списки участников — текстом «Имя» в строках-карточках. На референсе — стек кружков.
- **Как привести.** Реализовать стек аватарок (см. дизайн-систему). Для отказников — `grayscale + opacity-70 + крестик`. Для согласившихся — зелёная галочка.

### 4.6 Карточка «Площадка — расходы» отдельно от других управленческих действий
- **Проблема.** [event/[eventId]/page.tsx:738–778](../src/app/(app)/team/[id]/events/[eventId]/page.tsx#L738) — отдельная толстая карточка только для venue cost.
- **Как привести.** Объединить «Площадка», «Финансы», «Завершить/Отменить событие» в одну секцию «Управление» (список-строки + bottom action bar для деструктивных действий). См. дизайн-систему «Список действий».

### 4.7 Кнопки голосования вне карточки
- **Проблема.** «Приду / Не приду» — два больших оливковых рилла без подписи статуса.
- **Как привести.** Обернуть в карточку «Ваш ответ». Если не голосовал — обе кнопки secondary + строка «Вы ещё не ответили». Если проголосовал — выбранная primary, другая secondary.

### 4.8 «Сдал? / Был?» как контрол игрока конкурируют с организаторской таблицей
- **Проблема.** SelfMarkSection и OrganizerAttendanceList — два разных способа показать одно и то же. Игрок видит свою отметку отдельно, ниже список со всеми. У организатора нет своего «приду/не приду».
- **Как привести.** В деталях completed-события: показывать единый список участников с возможностью переключать «был/не был» только если ты сам этот пользователь или организатор. Раздел «Ваша отметка» убрать как отдельную секцию — отметка читается из строки в списке.

### 4.9 Bottom action bar отсутствует
- **Проблема.** Кнопки «Завершить / Отменить» — обычные в потоке контента, в самом низу содержимого, легко проскроллить.
- **Как привести.** Sticky bottom bar (`shadow-pop`), на экранах события скрыть BottomTabs (typing-логика уже есть в `BottomTabs.tsx` — добавить флаг для перекрытия action-баром).

## 5. Состав (`/team/[id]/roster`)

### 5.1 Нет верхней сводки
- **Проблема.** [roster/page.tsx](../src/app/(app)/team/[id]/roster/page.tsx) сразу показывает группы. Никаких метрик.

- **Файлы.** [team/[id]/roster/page.tsx](../src/app/(app)/team/[id]/roster/page.tsx). API: возможно расширить `GET /api/teams/[id]` с агрегатами по позициям.

### 5.2 Нет фильтр-чипов по позициям
- **Проблема.** Группы только «Организаторы» / «Игроки». Нет переключения по позиции/роли.
- **Как привести.** Чип-ряд: Все / Вратари / Защитники / Полузащ. / Нападающие. По умолчанию «Все».

### 5.3 Нет аватарок и мини-баров

### 5.4 Группы не по «ядро/резерв»
- **Проблема.** Только организатор/игрок. На референсе — «ЯДРО · 9», «РЕЗЕРВ · 3» (бизнес-смысл).
- **Как привести.** Это уже отдельная фича — пометить как post-MVP, либо оставить группировку по ролям, но с эйбрау-стилем «ОРГАНИЗАТОРЫ · N» в primary-цвете.

### 5.5 Кнопки «Организатор / Удалить» — на каждой строке, перегружают
- **Проблема.** [roster/page.tsx:188–209](../src/app/(app)/team/[id]/roster/page.tsx#L188) — две оливковые пилюли возле каждого игрока.
- **Как привести.** Спрятать в bottom-sheet, открывающийся по тапу на строку (для организатора). Сейчас уже есть `PlayerCard` — расширить его действиями «Сделать организатором» / «Удалить из команды».

### 6.5 DepositModal как bottom-sheet — ОК, но стиль формы старый

## 8. Профиль игрока (свой и чужой)

### 8.3 Inline-редактирование без визуального аффорданса

## 9. Общие переиспользуемые компоненты

### 9.1 Нет компонентов `Card`, `StatCard`, `Pill`, `Avatar`, `IconButton`, `BottomActionBar`
- **Проблема.** Все карточки и кнопки скопированы Tailwind-классами в каждом файле.
- **Почему плохо.** Любая правка дизайн-системы → изменение в 30 файлах.
- **Как привести.** В итерации 24 создать `src/components/ui/`:
  - `Card.tsx`, `StatCard.tsx`, `MiniBar.tsx`, `Avatar.tsx`, `AvatarStack.tsx`,
  - `Pill.tsx` (variant: filter/status/role), `Button.tsx` (variant: primary/secondary/danger),
  - `IconButton.tsx`, `BottomActionBar.tsx`, `EmptyState.tsx`, `SectionEyebrow.tsx`,
  - `PhotoBanner.tsx`, `ScreenHeader.tsx`.
- **Файлы.** Новые в `src/components/ui/`. Постепенно мигрировать остальные экраны.

### 9.2 BottomTabs не учитывает sticky bottom action
- **Проблема.** [BottomTabs.tsx](../src/components/BottomTabs.tsx) всегда виден, кроме момента ввода. Action bar события встанет поверх.
- **Как привести.** Добавить контекст `<UIProvider>` с `hideBottomTabs(true)` для экранов, где есть свой sticky bar; или менять через layout-флаг.

### 9.3 Skeleton использует `bg-border`, не `bg-background-muted`
- **Проблема.** Скелетоны слишком серые/контрастные, бросаются в глаза.
- **Как привести.** Использовать `bg-background-muted` (или новый `--skeleton`).
- **Файлы.** [src/components/Skeleton.tsx](../src/components/Skeleton.tsx).

### 9.4 BackButton встроен в hero
- **Проблема.** [BackButton.tsx](../src/components/BackButton.tsx) сейчас полупрозрачный (`bg-black/40`) — рассчитан на тёмный hero. После убирания hero его нужно перестилить под светлый фон (`bg-background-card shadow-card`).
- **Как привести.** Добавить вариант `kind="light" | "on-photo"`. По умолчанию light (для светлых хедеров), `on-photo` — для фото-баннеров.
