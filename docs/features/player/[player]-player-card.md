# Карточка игрока (TeamPlayerSheet)

Bottom-sheet с профилем игрока в контексте конкретной команды: надёжность, финансы, последние события и actions организатора. Дизайн на токенах v2 (`--ink-*`, `--green-*`, `--danger`).

## Где открывается

- Таб «Состав» (`/team/[id]/roster`) — клик по строке игрока
- Таб «Финансы» (`/team/[id]/finances`) — клик по строке должника или переплатившего (открывается с `initialSection="finances"`)
- Главная команда (`/team/[id]`) — клик по компактной строке участника

Один компонент `TeamPlayerSheet` (`src/components/team/TeamPlayerSheet.tsx`).

## Header

- **Аватар** 64×64 круглый, обводка `--ink-200`. Без фото — `--ink-100` фон + инициал `--ink-500`
- **Имя** — Oswald 22px uppercase, `--ink-900`
- **Под именем (flex-wrap)**:
  - Если `member.role === "organizer"` — пилюля «Организатор», `--green-50` фон + `--green-700` текст + иконка короны, 11px uppercase
  - Если есть `city` — `📍 city` 13px `--ink-500`
- **`RatingRing`** 56px справа от данных (toкены v2, цвет числа от tier)
- **`PositionTag`** теги под шапкой (`--pos-{kind}-bg/fg`)
- **Кнопка «Открыть профиль»** — secondary, `h-11 rounded-[12px]`, `--bg-secondary` + `--ink-200` border, `--ink-900` текст. Ведёт на `/players/[id]`

## Секции (аккордеон)

Каждая секция — белая карточка `--card` с границей `--ink-100`, тап по заголовку раскрывает. Внутри раскрытой части есть верхняя разделительная линия `--ink-100`.

### Надёжность · в этой команде

Peek (свёрнуто): «N неприходов · M отмен» либо «Без нарушений», справа `{N}%` (`--green-700` ≥90, `--ink-900` ≥70, `--danger` иначе).

Раскрыто:
- Большое число `{N}%` + лейбл («Стабильный»/«Надёжный»/«Средняя»/«Низкая»)
- 2 mini-stat (`--bg-secondary` фон): Неприходы / Отмены
- Список последних событий (`--bg-secondary` фон списка): точка-маркер + лейбл («Сыграл матч» / «Не пришёл» / «Отменил» / «Записался — Тренировка») + дата справа + chevron

API: `GET /api/teams/[id]/members/[userId]/reliability?userId=requesterId`

### Финансы · в этой команде

Видна, если просмотрщик — организатор или сам игрок.

Peek: «Сдал X из Y ₸» + сальдо `±N ₸` (`--green-700` — переплата, `--danger` — долг).

Раскрыто:
- Eyebrow «Переплата»/«Долг» + Oswald 28px число
- 2 mini-stat: Должен сдать / Сдал
- История транзакций (до 8): label + сумма (`--green-700` — депозит). Клик по записи с `event_id` ведёт на событие

API: `GET /api/teams/[id]/members/[userId]/finances?userId=requesterId`

### Игровая статистика

Заглушка «🏗 Скоро — голы, передачи, MVP».

## Actions

Видны только организаторам (если просмотрщик — не сам игрок).

- **«Сделать организатором»** — secondary, `--bg-secondary` + `--ink-200` border, `--ink-900` текст; `PATCH /api/teams/[id]/members/[memberId]`
- **«Удалить из команды»** — `--danger-soft` фон + `--danger` текст; `DELETE /api/teams/[id]/members/[memberId]?userId=`

Если просмотрщик смотрит свой профиль — видна **«Покинуть команду»**: `--danger-soft` + `--danger` + иконка logout. Тап → инлайн-подтверждение (отмена + filled `--danger` Выйти) → `POST /api/teams/[id]/leave`.

Ошибки экшенов выводятся как pill `--danger-soft` + `--danger` над кнопками.

## UI

- Бэкдроп `rgba(0,0,0,0.4)`, тап закрывает
- Sheet: `--card` фон, `rounded-t-[24px]`, max-h `88vh`, скролл внутри
- Drag-handle 36×4 (`--ink-300`) + крестик 28×28 (`--ink-100` фон, `--ink-700` stroke)
