# Карточка игрока

Bottom sheet с историей посещений и финансовым балансом игрока внутри конкретной команды.

## Где открывается

- Таб «Состав» — клик по строке игрока (только организатор)
- Таб «Финансы» — клик по строке должника или переплатившего

Один компонент `PlayerCard`, общий для обоих сценариев.

## Содержимое

- **Имя и город** игрока
- **Сальдо**: разница между сданными и ожидаемыми суммами по completed-событиям
  - `> 0` — команда должна игроку (переплата)
  - `< 0` — игрок должен команде (долг)
  - `= 0` — расчёты сошлись
- **Итоги**: всего сдано / ожидалось / баланс
- **История** по completed-событиям:
  - Был, сдал X ₽
  - Был, должен Y ₽
  - Не был
  - Тип события и дата

## API

`GET /api/teams/[id]/members/[userId]/finances?userId=requesterId`

Авторизация: организатор может смотреть карточку любого участника, остальные — только свою.

Возвращает: `user`, `totals { expected, paid, balance }`, `history[]` по completed-событиям.

## UI

- Bottom sheet: `bg-background-card rounded-t-xl shadow-pop`
- Хедер: `Avatar md` + имя (`text-[17px] font-semibold`) + город
- Сальдо: число `text-[28px] tabular-nums`, `text-primary` если ≥0, `text-danger` если <0
- История: `divide-y divide-border`, депозит помечается `Pill variant="role"`
- `organizerActions` slot — кнопки через `Button` (promote/remove) снизу
- Закрывается кликом по `bg-background-overlay` overlay или крестику
