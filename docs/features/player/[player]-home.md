# Главная (`/home`)

## Структура экрана

### Приветствие

Простой светлый заголовок без тёмного блока:
- Имя пользователя — `text-[28px] font-bold`
- Город — `text-[13px] text-foreground-secondary` (если задан)

### Ближайшее событие

Карточка ближайшего будущего события из команд пользователя или событий, на которые он записался (`vote = yes`). Данные из `GET /api/users/[id]/next-event`.

Структура блока:

1. **PhotoBanner** — градиент-плейсхолдер (фото площадки — post-MVP). Поверх: статус-пилюли (тип события + «Запланировано»), внизу — дата, площадка, команда. Клик → `/team/[teamId]/events/[eventId]`.

2. **Триплет-метрика** — карточка с тремя колонками:
   - Взнос (`price_per_player ₸`)
   - Мин. игроков
   - Участники `X / мин` + тонкий прогресс-бар

3. **Карточка голосования** — показывается всегда для запланированных событий:
   - Если юзер ещё не голосовал: обе кнопки secondary + подпись «Вы ещё не ответили»
   - Если проголосовал: выбранная кнопка выделена (primary / danger), подпись «Вы идёте» / «Вы не идёте»
   - Голосование отправляется в `POST /api/teams/[teamId]/events/[eventId]/vote`

Если нет предстоящих событий — блок не отображается (вместо него EmptyState при отсутствии команд).

### Быстрые действия

Сетка `grid-cols-2`:
- **Найти игру** → `/search` — primary (зелёный фон)
- **Команды** → `/teams` — secondary (белый фон с border)

### Мои команды

Компактный список в `Card`. Отображается только если команды есть.

Каждая строка:
- Имя команды + город · спорт
- Пилюля роли: «Организатор» (`role`) / «Игрок» (`filterActive`)
- Клик → `/team/[id]`

Разделение строк — `divide-y divide-border`.

### Пустое состояние

Показывается если нет ни команд, ни ближайших событий — `EmptyState` с иконкой и CTA «Найти игру» → `/search`.

## API

### `GET /api/users/[id]/next-event`

Ближайшее запланированное событие пользователя.

**Логика:**
1. Команды пользователя из `team_memberships`
2. Проголосованные события из `event_attendances` где `vote = yes`
3. Ближайшее событие с `status = planned` и `date > now()` из обоих источников
4. Счётчик `yes_count` + текущий голос пользователя `user_vote`

**Ответ:**
```json
{
  "event": {
    "id": "...",
    "type": "game",
    "date": "2026-04-25T19:00:00Z",
    "team_id": "...",
    "price_per_player": 2000,
    "min_players": 10,
    "is_public": false,
    "team": { "id": "...", "name": "ЦСКА" },
    "venue": { "id": "...", "name": "Поле №1", "address": "..." },
    "yes_count": 8,
    "user_vote": "yes"
  }
}
```

`event: null` — нет предстоящих событий. `user_vote: null` — пользователь не голосовал.
