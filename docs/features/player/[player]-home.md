# Главная (`/home`)

## Структура экрана

### Хедер приветствия

Тёмный блок с именем пользователя (uppercase, font Oswald). Отображается всегда.

### Ближайшее событие

Карточка ближайшего будущего события из команд пользователя или событий, на которые он записался (`vote = yes`). Данные из `GET /api/users/[id]/next-event`.

Карточка содержит:
- Бейдж типа события (Игра / Тренировка / Сбор / Другое)
- Название команды
- Дата и время (локаль `ru-RU`)
- Название площадки (если указана)
- Количество записавшихся (`vote = yes`)
- Клик → `/team/[teamId]/events/[eventId]`

Empty state: «Нет предстоящих событий» — серый блок.

### Быстрые действия

Два кнопки в сетке 2 колонки:
- **Найти игру** → `/search`
- **Команды** → `/teams`

### Мои команды

Компактный список команд пользователя. Отображается только если команды есть.
Каждая строка: название + роль (Организатор / Игрок) + город · спорт.
Клик → `/team/[id]`.

Если команд нет и нет ближайшего события — единый empty state «Вступи в команду, чтобы видеть события».

## API

`GET /api/users/[id]/next-event` — ближайшее запланированное событие пользователя.

Логика выборки:
1. Получить team_id из `team_memberships` пользователя
2. Получить event_id из `event_attendances` где `vote = yes`
3. Найти ближайшее событие с `status = planned` и `date > now()` из обоих источников
4. Вернуть с полями: id, type, date, team_id, team (name), venue (name, address), yes_count

Ответ:
```json
{
  "event": {
    "id": "...",
    "type": "game",
    "date": "2026-04-25T19:00:00Z",
    "team_id": "...",
    "team": { "id": "...", "name": "Первый" },
    "venue": { "id": "...", "name": "Поле №1", "address": "..." },
    "yes_count": 8
  }
}
```

`event: null` если нет предстоящих событий.
