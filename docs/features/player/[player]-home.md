# Главная (`/home`)

## Структура экрана

Блоки идут сверху вниз в следующем порядке.

---

### 1. Hero — ближайшее событие

**Кому:** все пользователи.

Hero-зона — зелёный фон с диагональной текстурой, скруглением снизу 28px. Внутри:

- **Header:** «Привет, [имя]» + bell-иконка справа. Red-dot на bell, если у пользователя есть pending-заявки в команды-организатор. Тап на bell → bottom-sheet со списком заявок по командам (тап на пункт → заявки команды).
- **Тёмная карточка события** (gray-900):
  - Фото-баннер 140px, градиент-overlay
  - Бейджи типа события (белый) и статуса (glass) — top-left
  - Countdown «через 2д» (Oswald) — bottom-right на фото
  - Заголовок команды — Oswald uppercase 22px, белый
  - 3 чипа: время, площадка, цена
  - Явка-bignum: «7 / 10» (Oswald) + 3-сегментный бар (yes/no/waiting) + легенда
  - RSVP-кнопки «Приду» (зелёная с галочкой если выбрано) / «Не приду» (transparent border)

Если событий нет — hero-зона показывается без карточки события (только header), под ней — EmptyState с CTA «Найти команду».

---

### 2. Заявки в мои команды

**Кому:** только организатор, только при наличии заявок.

Компактная карточка-сигнал. Если заявок нет — блок не отображается.

Содержимое:
- Счётчик входящих заявок: «N игроков хотят вступить»
- Если заявки в несколько команд — разбивка по командам
- CTA-кнопка «Посмотреть» → переход к заявкам соответствующей команды

---

### 3. Быстрые действия

**Кому:** все пользователи.

Сетка `grid-cols-2`, две кнопки:
- **Найти событие** — primary (зелёный фон) → `/search` (вкладка «События»)
- **Найти команду** — secondary → `/search` (вкладка «Команды»)

---

### 4. Пульс команд

**Кому:** только организатор.

Горизонтальный скролл карточек — по одной на каждую команду, где пользователь является организатором.

Каждая карточка содержит:
- Название команды
- Ближайшее событие: дата или «нет запланированных»
- Явка на ближайшее событие: «N / min» + прогресс-бар (цвет: зелёный если достаточно, warning если дефицит)
- Счётчик долгов: «долги: N игроков» (только если > 0, красный)
- Счётчик заявок: «N заявок» (только если > 0)

Тап на карточку → переход в команду.

---

### 5. Расписание

**Кому:** все пользователи.

Заголовок «Расписание» + ссылка «Все события» справа.

Список из 3 ближайших событий после hero-события. Каждая строка:
- Дата + день недели
- Название команды
- Тип события (пилюля)
- Статус явки пользователя: «иду» / «не иду» / «не ответил»

Тап на строку → событие.

**«Все события»** → `/search` с предустановленным фильтром «события моих команд» (где пользователь орг или игрок).

---

## Источники данных

| Блок | API |
|------|-----|
| Hero | `GET /api/users/[id]/next-event` |
| Заявки | `GET /api/users/[id]/pending-requests` |
| Пульс команд | `GET /api/users/[id]/teams-pulse` |
| Расписание | `GET /api/users/[id]/schedule?limit=3&offset=1` |
