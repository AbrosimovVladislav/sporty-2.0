# Поиск

Универсальный публичный каталог. Один путь ко всем 4 сущностям.

## Архитектура

`/search` — хаб, редиректит на `/search/events`. Каждый саб-таб — отдельный роут с собственным `PageHeader` (зелёный hero + 0–2 stats), под ним `SearchSubnav` (chip-row), затем стандартный listing-toolbar (search / sort / опц. pills / chips) и список.

Саб-табы (в порядке отображения):
1. [`/search/events`]([search]-events.md) — публичные события
2. `/search/teams` — каталог команд
3. `/search/players` — каталог игроков
4. `/search/venues` — каталог площадок

Дефолт — события (наиболее универсальный интент: «найти, куда прийти»).

## Унифицированный паттерн листинг-страницы

Все 4 саб-таба следуют одной структуре:

1. **`PageHeader`** + 0–2 `HeaderStat`:
   - События: «Всего» + «На неделе»
   - Команды: «Всего» + «Ищут игроков»
   - Игроки: «Всего» + «Ищут команду»
   - Площадки: без stat-карточек
2. **`SearchSubnav`** — chip-row саб-табов
3. **`ListSearchBar`** — поиск по имени/полю
4. **`ListMeta`** — sort-dropdown справа. Счётчика «Найдено N» больше нет — он переехал в эйбрау «Результаты · N» над списком (50.1)
5. **`FilterPills`** (опц.) — быстрые чипы: тип события / позиция игрока (5.0–5.5: на `/search/players` чипы позиций — единственная точка управления, в `PlayerFiltersSheet` поля «Позиция» нет)
6. **`ActiveFilterChips`** — применённые фильтры из шита
7. **Эйбрау «Результаты · N»** + сам список

### Сортировки (50.3)

| Таб | Default | Остальные |
|---|---|---|
| События | Сначала ближайшие (`date_asc`) | Сначала далёкие, По цене (дешевле) |
| Команды | Сначала новые (`created_desc`) | По названию (А-Я) |
| Игроки | По уровню (`skill`) | По имени (А-Я), Недавно зарегистрированные |
| Площадки | По названию (А-Я) на бэкенде | dropdown скрыт — одна опция |

### Позиции игроков

Везде, где отображаются позиции (`PlayerListRow`, `TeamPlayerSheet`), используется компонент [`PositionChip`](../../../src/components/PositionChip.tsx) — цветные чипы с сокращениями ВРТ/ЗАЩ/ПЗЩ/НАП. Цвета: `amber` / `blue` / `emerald` / `red`. Поддерживается `tone: 'light' | 'dark'`.

Multi-position игрок попадает во все свои чипы — backend-фильтр идёт через `contains("position", [pos])` (50.4, реализовано ещё в 47.3).

## Принцип «Моё ≠ Публичное»

«Поиск» — это **только публичные сущности**. Никаких «моих» режимов и scope-переключателей. Если пользователь хочет добраться до своей команды — он идёт через таб «Моя команда» (`/teams` → `/team/[id]`). Поиск — для discovery, не операций.

В строках поиска допустимы информационные бейджи о связи пользователя со строкой («Моя команда», «Капитан», «Я в составе»), но они не меняют скоуп списка — только подсказывают.

## Что осознанно не делаем

- Нет одного «общего поискового поля по всем 4 сущностям». Это упрощение каталога без бизнес-ценности на MVP — пользователю всё равно нужно понять, что он ищет
- Нет state-saved между сессиями (последний посещённый сабтаб). URL — единственный источник истины
- Нет доп. action-кнопок в зелёном хедере поиска (вроде «+ Создать»). Создание команды живёт в `/teams` и в team-switcher, не в discovery
