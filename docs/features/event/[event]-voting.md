# Голосование

Участник или гость открывает карточку события и отмечает:

- **Приду** — подтверждает участие
- **Не приду** — отказывается

Голос можно изменить до завершения события (пока статус `planned`).

## Кто может голосовать

- **Участник команды (player/organizer):** может голосовать на любом событии команды
- **Гость:** может голосовать только на событиях с `is_public = true`

## UI

Две кнопки-тогглы на карточке события. Активная кнопка подсвечивается (зелёная — приду, красная — не приду). Кнопки отображаются для участников всегда, для гостей — только на публичных событиях.

## API

`POST /api/teams/[id]/events/[eventId]/vote` — body: `{ userId, vote: "yes" | "no" }`

Логика авторизации:
- Событие `is_public = true`: достаточно аутентификации (userId)
- Событие `is_public = false`: требуется членство в команде
