# Голосование

Участник или гость открывает карточку события и отмечает:

- **Приду** — подтверждает участие
- **Не приду** — отказывается

Голос можно изменить до завершения события (пока статус `planned`).

## Кто может голосовать

- **Любой аутентифицированный пользователь**, открывший страницу события (по прямой ссылке или из листингов).

`is_public` влияет только на видимость в листингах (см. [event]-detail.md), не на право голоса. Приватное событие = «по ссылке»: его не видно в поиске и на странице команды у гостей, но любой, кому организатор отправил ссылку, может ответить «Приду».

## UI

Карточка «Ваш ответ» на детальном экране события. Кнопки «Приду» (становится зелёной при выборе) и «Не приду» (становится красной при выборе). Под кнопками — строка статуса: «Вы ещё не ответили» / «Вы идёте» / «Вы не идёте».

После голосования на экране события обновляются секции «Придут (N)» и «Не придут (N)» со стеками аватарок.

## API

`POST /api/teams/[id]/events/[eventId]/vote` — body: `{ userId, vote: "yes" | "no" }`

Авторизация: достаточно `userId`. Событие должно существовать и иметь статус `planned`.
