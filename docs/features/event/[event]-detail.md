# Карточка события

Основной экран события (`/team/[id]/events/[eventId]`). Открывается по прямой ссылке для любого аутентифицированного пользователя — `is_public` влияет только на видимость в листингах, а не на доступ к самой странице или возможность голосовать. См. «Видимость и публичность».

## Структура экрана

### 1. Photo Banner

Вверху — `PhotoBanner` с градиентным плейсхолдером (фото площадки — post-MVP).
- Поверх слева: пилюля типа события (зелёная), пилюля статуса (белая полупрозрачная)
- Поверх снизу: дата + название площадки

### 2. Триплет-метрика

Карточка из трёх колонок: взнос / мин. игроков / участников (X/min + прогресс-бар).

### 3. Карточка «Ваш ответ» (planned)

Видна любому аутентифицированному пользователю. Кнопки «Приду» (primary при выборе) / «Не приду» (danger при выборе) + строка статуса под ними.

### 4–5. Придут (N) / Не придут (N)

Показываются для planned и cancelled событий. Каждая группа: `SectionEyebrow` + `AvatarStack` (max 4) + кнопка «Показать всех» при N > 4. Аватарки «не придут» — grayscale + opacity-70.

### 6. Участники (completed)

Единый список строк: аватар + имя + тоггл «Был» + тоггл «Сдал».
- Организатор: может переключать оба поля для любого участника
- Игрок: может переключать оба поля только в своей строке
- Остальные строки: read-only индикаторы

Подробнее: [[event]-attendance.md]([event]-attendance.md)

### 7. Управление (только организатор)

Карточка со списком строк:
- **Площадка**: стоимость + inline-редактирование по кнопке «Указать» / «Изменить»
- **Оплата площадки**: тоггл «Оплачено» (при venue_cost > 0)
- **Финансы**: для planned — ожидаемый сбор; для completed — собрано из ожидаемого

### 8. Bottom Action Bar (organizer + planned)

Sticky-полоса внизу: кнопки «Завершить» / «Отменить». При наличии этой полосы `BottomTabs` скрывается. Подробнее: [[event]-lifecycle.md]([event]-lifecycle.md)

## Доступность экрана

| Роль | Доступ |
|------|--------|
| Организатор | Полный |
| Игрок | Голосование, своя отметка |
| Гость (любое событие) | Голосование «Приду/Не приду» |

## Видимость и публичность

Флаг `is_public` управляет видимостью события в листингах:

- **`is_public = true` (публичное)**: событие видно в общем поиске (`/search/events`), на странице команды у гостей и в карточке площадки.
- **`is_public = false` (приватное)**: скрыто из всех листингов, доступно только по прямой ссылке. На карточке события и в строках списков (видимых членам команды) показывается значок 🔒 «Приватное».

Голосовать «Приду/Не приду» может любой аутентифицированный пользователь, открывший страницу события — это позволяет организатору приглашать сторонних игроков по ссылке без добавления в команду. Подробнее: [[event]-voting.md]([event]-voting.md).

Переключение `is_public` доступно организатору в секции «Управление» на детальном экране.
