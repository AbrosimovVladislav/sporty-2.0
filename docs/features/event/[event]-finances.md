# Финансы события

Финансовая сводка по событию. Видит только организатор.

## Показатели завершённого события

- **Были на событии** — количество подтверждённых участников (`attended_confirmed ?? attended`)
- **Ожидаемый сбор** — Σ `price_per_player` по подтверждённым участникам
- **Сдали деньги** — количество из подтверждённых, кто сдал (`paid_confirmed ?? paid`)
- **Фактический сбор** — Σ фактических оплат: `paid_amount`, либо `price_per_player` если флаг сдал и сумма не указана
- **Разница** — фактический − ожидаемый (профицит зелёным, дефицит красным)

## Площадка — расходы

Блок «Площадка — расходы» (организатор, любой статус события):

- **Стоимость** — `events.venue_cost`
- **Оплачено** — `events.venue_paid`
- **Остаток** — стоимость − оплачено

Поля редактируются inline через PATCH `/api/teams/[id]/events/[eventId]`.

## Принцип

Сбора денег через приложение нет — только учёт. Деньги передаются лично.

Кнопка «Сдал» по умолчанию ставит сумму = `price_per_player`. Кнопка «Сумма» рядом — открывает inline-input для произвольной суммы (профицит/дефицит). Сохраняется в `event_attendances.paid_amount`. Снятие отметки «Сдал» обнуляет `paid_amount`.

## Предварительная оценка (planned)

Для запланированных событий организатор видит упрощённый блок:
- Ожидаемый сбор = проголосовавшие «приду» × цена
