# Команда

Команда — центральная сущность приложения. Все активности (события, финансы, состав) живут внутри команды.

## Модель ролей

Роль пользователя — свойство его связи с командой (`TeamMembership.role`), а не свойство самого пользователя. Один человек может быть игроком в одной команде и организатором в другой.

### Роли

- **player** — участник команды, голосует за события, видит состав и расписание
- **organizer** — расширенные права: управление составом, создание событий, финансы, обработка заявок

В одной команде может быть **несколько организаторов** с одинаковыми правами. Первый организатор — создатель команды.

### Как стать организатором

1. **Создать команду** — пользователь становится первым организатором
2. **Получить промоут** от существующего организатора — `TeamMembership.role` меняется с `player` на `organizer`

Заявка «стать организатором извне» — отложена за MVP, см. [post-mvp.md](../../post-mvp.md).

## Загрузка лого

Организатор видит иконку-камеру поверх аватара команды в шапке. Тап → системный file-picker (до 2 МБ). Загружается через `POST /api/teams/[id]/logo` в Supabase Storage (`team-logos/`), после чего `logo_url` команды обновляется автоматически.

## Экран команды (`/team/[id]`)

Единый роут с разным содержимым в зависимости от отношения пользователя к команде:

1. **Не-участник** — публичный вид с кнопкой заявки → [[team]-public-view.md]([team]-public-view.md)
2. **Игрок** — полный доступ без финансов и управления
3. **Организатор** — всё + финансы + управление

Суб-навигация: Главная / Состав / События для всех ролей; Финансы — только для организатора.
