# Таб «Состав»

Маршрут: `/team/[id]/roster`

Использует общую шапку и саб-навигацию из `team/[id]/layout.tsx` (зелёный `PageHeader` + underline-табы). Кастомный NavBar удалён.

## Структура экрана

### Тулбар

- `ListSearchBar` — поиск по имени (фильтрация на клиенте, список обычно 10–30 человек)
- `ListMeta` — «N игроков» слева + sort-dropdown справа
- `FilterPills` — позиции: Все / ВРТ / ЗАЩ / ПЗЩ / НАП

### Список участников

`PlayerListRow` (тот же компонент, что на `/search/players`):
- Avatar (инициалы, `Avatar size="md"`)
- Имя + бейдж «ОРГАНИЗАТОР» (только у организаторов, `RoleBadge` серый)
- Мета: позиция · город
- Правый слот: bar надёжности (показывает «—» если нет данных, т.к. `TeamMember` не содержит `reliability`)

Тап на строку → открывает `PlayerSheet` (bottom-sheet).

### Сортировка

| Значение | Поведение |
|---|---|
| `skill` (default) | По уровню мастерства, убывание |
| `recent` | По дате вступления, убывание |
| `organizers` | Организаторы сверху, затем игроки |

### Пустое состояние

- При поиске/фильтре без результатов: «Никого не нашлось»
- Без фильтров, пустой состав: «Состав пуст»

## PlayerSheet (bottom-sheet)

Bottom-sheet с карточкой игрока:
- `Avatar size="lg"` + имя + позиция + бейдж роли
- Статбоксы: уровень (N/5) и город
- Кнопка «Открыть профиль» → `/players/[userId]`

Организатор (не сам себе):
- «Сделать организатором» — только для игроков; `PATCH /api/teams/[id]/members/[memberId]`
- «Удалить из команды» — `DELETE /api/teams/[id]/members/[memberId]?userId=`

Действия закрывают шит и вызывают `reload()` контекста.

## API

Данные берутся из `TeamContext` (загружаются при открытии команды).

`GET /api/teams/[id]` возвращает `members` с полями `user.position`, `user.skill_level`, `user.city`, `joined_at`.

Заявки на вступление управляются через bell-иконку в общей шапке (`TeamRequestsSheet`), кнопки на странице состава нет.
