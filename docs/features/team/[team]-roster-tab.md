# Таб «Состав»

Маршрут: `/team/[id]/roster`

Использует общую шапку и саб-навигацию из `team/[id]/layout.tsx` (зелёный `PageHeader` + underline-табы). Кастомный NavBar удалён.

## Структура экрана

### Тулбар

- `ListSearchBar` — поиск по имени (фильтрация на клиенте, список обычно 10–30 человек)
- `ListMeta` — «N игроков» слева + sort-dropdown справа
- `FilterPills` — позиции: Все / ВРТ / ЗАЩ / ПЗЩ / НАП

### Список участников

`PlayerListRow` (тот же компонент, что на `/search/players`):
- Avatar (инициалы, `Avatar size="md"`)
- Имя + бейдж «ОРГАНИЗАТОР» (только у организаторов, `RoleBadge` серый)
- Мета: позиция · город
- Правый слот: bar надёжности (показывает «—» если нет данных, т.к. `TeamMember` не содержит `reliability`)

Тап на строку → открывает `TeamPlayerSheet` (bottom-sheet, единая карточка игрока внутри команды).

### Сортировка

| Значение | Поведение |
|---|---|
| `skill` (default) | По уровню мастерства, убывание |
| `recent` | По дате вступления, убывание |
| `organizers` | Организаторы сверху, затем игроки |

### Пустое состояние

- При поиске/фильтре без результатов: «Никого не нашлось»
- Без фильтров, пустой состав: «Состав пуст»

## TeamPlayerSheet (bottom-sheet)

Единый компонент карточки игрока в команде — открывается из всех точек внутри `/team/[id]/*` (состав, лидеры на главной, должники в финансах в будущем).

**Шапка (всегда видима):** `Avatar size="lg"` + имя + бейдж «Организатор» (если орг) + чипы (амплуа из массива позиций, уровень N/5, город) + кнопка «Открыть профиль» → `/players/[userId]`.

**Аккордеоны** (раскрываются по тапу, lazy-load):

1. **Надёжность · в этой команде** — `GET /api/teams/[id]/members/[userId]/reliability?userId=requesterId`. Поля: `totals.reliability` (%), `played`/`votedYes`, `noShow`, `cancelled`, `recentEvents[10]` со статусом «Был / Не пришёл / Отменил». Auth: requester должен быть участником команды
2. **Финансы · в этой команде** — `GET /api/teams/[id]/members/[userId]/finances?userId=requesterId`. Видно только organizer/self. Сальдо (долг/переплата), expected/paid, история транзакций
3. **Игровая статистика** — placeholder до появления трекинга

**Действия** (внизу):
- Organizer (не сам себе): «Сделать организатором» (только для player) `PATCH /api/teams/[id]/members/[memberId]`; «Удалить из команды» `DELETE /api/teams/[id]/members/[memberId]?userId=`
- Self (любая роль): «Покинуть команду» с двухшаговым подтверждением `POST /api/teams/[id]/leave` → редирект `/home`. Последний организатор получает ошибку «Назначьте другого организатора перед выходом»

Действия закрывают шит и вызывают `reload()` контекста (или редирект на `/home` при выходе).

## API

Данные берутся из `TeamContext` (загружаются при открытии команды).

`GET /api/teams/[id]` возвращает `members` с полями `user.position`, `user.skill_level`, `user.city`, `joined_at`.

`POST /api/teams/[id]/leave` — тело `{ userId }`. Разрешено любому участнику. Организатор-одиночка получает ошибку 400.

Заявки на вступление управляются через шестерёнку в шапке (`/team/[id]/settings` → раздел «Заявки и приглашения»), кнопки на странице состава нет.
