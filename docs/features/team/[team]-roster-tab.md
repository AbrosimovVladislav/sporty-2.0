# Таб «Состав»

Маршрут: `/team/[id]/roster`

Использует общую шапку и саб-навигацию из `team/[id]/layout.tsx` (зелёный `PageHeader` + underline-табы). Кастомный NavBar удалён.

## Структура экрана

### Тулбар

- `ListSearchBar` — поиск по имени + кнопка-воронка (`onFilterClick`) с бейджем активных фильтров. Фильтрация на клиенте, список обычно 10–30 человек
- `ListMeta` — «N игроков» слева + sort-dropdown справа
- Воронка открывает bottom-sheet «Фильтры» с `SheetChipGroup` по позиции (single-select). Внизу — кнопки «Сбросить» / «Применить»

### Список участников

`PlayerListRow` (тот же компонент, что на `/search/players`):
- Avatar (инициалы, `Avatar size="md"`)
- Имя + бейдж «ОРГАНИЗАТОР» (только у организаторов, `RoleBadge` серый)
- Мета: позиция · город
- Правый слот: bar надёжности (показывает «—» если нет данных, т.к. `TeamMember` не содержит `reliability`)

Тап на строку → открывает `TeamPlayerSheet` (bottom-sheet, единая карточка игрока внутри команды).

### Сортировка

| Значение | Поведение |
|---|---|
| `skill` (default) | По уровню мастерства, убывание |
| `recent` | По дате вступления, убывание |
| `organizers` | Организаторы сверху, затем игроки |

### Пустое состояние

- При поиске/фильтре без результатов: «Никого не нашлось»
- Без фильтров, пустой состав: «Состав пуст»

## TeamPlayerSheet (bottom-sheet)

Единый компонент карточки игрока в команде — открывается из всех точек внутри `/team/[id]/*` (состав, лидеры на главной, должники в финансах в будущем). Дизайн на токенах v2 (`--ink-*`, `--green-*`, `--danger`).

**Шапка (всегда видима):** аватар (64px) + имя (display 22px) + пилюля «Организатор» (зелёный soft) + город (с пином) + `RatingRing` (56px) справа. Под шапкой — `PositionTag` тегами. Ниже — кнопка-ссылка «Открыть профиль» (secondary, `--bg-secondary` + `--ink-200` border) → `/players/[userId]`.

**Аккордеоны.** Каждый — белая карточка с границей `--ink-100`. Сводка предзагружается, поэтому в свёрнутом виде сразу виден ключевой показатель справа:

1. **Надёжность · в этой команде** (`GET /api/teams/[id]/members/[userId]/reliability?userId=requesterId`)
   - В свёрнутом виде: процент надёжности (`--green-700` ≥90, `--ink-900` ≥70, `--danger` иначе) + подзаголовок `N неприходов · M отмен` либо «Без нарушений»
   - В раскрытом: большая цифра + лейбл состояния, мини-карточки `Неприходы` / `Отмены` (`--bg-secondary`), список последних 10 событий со статусом и датой. **Тап по строке события → `/team/[id]/events/[eventId]`**
   - Auth: requester должен быть участником команды
2. **Финансы · в этой команде** (`GET /api/teams/[id]/members/[userId]/finances?userId=requesterId`)
   - Видно только organizer/self
   - В свёрнутом виде: сальдо (`+X ₸` `--green-700` или `−X ₸` `--danger`) + подзаголовок `Сдал X из Y ₸`
   - В раскрытом: долг/переплата крупно, мини-карточки `Должен сдать` / `Сдал`, история транзакций. Записи привязанные к событию кликабельны → переход к событию (`event_id` приходит в ответе)
3. **Игровая статистика** — placeholder до появления трекинга

**Действия** (внизу):
- Organizer (не сам себе): «Сделать организатором» (secondary, только для player) `PATCH /api/teams/[id]/members/[memberId]`; «Удалить из команды» (`--danger-soft`) `DELETE /api/teams/[id]/members/[memberId]?userId=`
- Self (любая роль): «Покинуть команду» (`--danger-soft` + иконка logout) с двухшаговым подтверждением `POST /api/teams/[id]/leave` → редирект `/home`. Последний организатор получает ошибку «Назначьте другого организатора перед выходом»
- Ошибки экшенов — pill в `--danger-soft` + `--danger`

Действия закрывают шит и вызывают `reload()` контекста (или редирект на `/home` при выходе).

## API

Данные берутся из `TeamContext` (загружаются при открытии команды).

`GET /api/teams/[id]` возвращает `members` с полями `user.position`, `user.skill_level`, `user.city`, `joined_at`.

`POST /api/teams/[id]/leave` — тело `{ userId }`. Разрешено любому участнику. Организатор-одиночка получает ошибку 400.

Заявки на вступление управляются через шестерёнку в шапке (`/team/[id]/settings` → раздел «Заявки и приглашения»), кнопки на странице состава нет.
