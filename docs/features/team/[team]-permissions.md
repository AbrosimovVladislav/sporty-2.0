# Права и видимость

Три роли: `guest` (не состоит в команде), `player`, `organizer`.

Сводная таблица: что видит каждая роль на экране команды.

| Что | Гость | Игрок | Организатор |
|---|---|---|---|
| Название, город, спорт | + | + | + |
| Описание команды | + | + | + |
| Количество игроков | + | + | + |
| Состав (имена игроков) | + | + | + |
| Список событий | только публичные | все | все |
| Ближайшее событие (главная) | только публичные | все | все |
| Голосование за событие | только публичные | все planned | все planned |
| Сумма к оплате за событие | - | + (своя) | + |
| Финансовый баланс команды | - | - | + |
| Создание события | - | - | + |
| Управление составом | - | - | + |
| Обработка заявок | - | - | + |
| Промоут игрока в организаторы | - | - | + |
| Подать заявку на вступление | + | - | - |

## Суб-навигация по роли

- **Гость:** Главная, Состав, События
- **Игрок:** Главная, Состав, События
- **Организатор:** Главная, Состав, События, Финансы

## Фильтрация событий

`GET /api/teams/[id]/events` проверяет членство по `userId`:
- Участник (player/organizer): возвращает все события команды
- Гость или без userId: возвращает только события с `is_public = true`
