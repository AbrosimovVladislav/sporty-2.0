# Таб «События»

Маршрут: `/team/[id]/events`

Использует общую шапку и саб-навигацию из `team/[id]/layout.tsx` (зелёный `PageHeader` + underline-табы). Контент в стандартном `px-4 py-4` wrapper'е.

## Структура экрана

### Тулбар

`FilterPills` — фильтр по типу события (клиентский): Все / Игра / Трен. / Сбор / Другое

### Секции

Два блока с eyebrow-заголовком:
- **ПРЕДСТОЯЩИЕ · N** — события со `status = "planned"`
- **ПРОШЕДШИЕ · N** — события со `status = "completed"` или `"cancelled"`

Каждое событие — строка `EventListRow`:
- Дата-блок (день + месяц)
- Тип события (Игра / Тренировка / Сбор / Другое) как основной заголовок
- Подзаголовок: тип (полностью) · время · площадка
- Правый слот: количество «придут» + цена

Тап → `/team/[id]/events/[eventId]`.

### Пустые состояния

- «Нет событий этого типа» — при активном фильтре
- «Событий пока нет» — список пустой

## Создание события (организатор)

Зелёный FAB «+» (fixed bottom-right, `z-20`) — только для организатора.

Тап → `EventCreateSheet` (bottom-sheet до 92dvh, скроллируемая форма):

| Поле | Компонент |
|---|---|
| Тип события | `SheetChipGroup` (Игра / Тренировка / Сбор / Другое) |
| Дата / Время | native `<input type="date/time">` |
| Цена с игрока, Мин. игроков | number inputs |
| Площадка | native `<select>` (Не указана / список / + Новая) |
| Новая площадка | поля Название, Адрес, `DistrictSelect` |
| Стоимость площадки | number input (при выборе площадки) |
| Описание | `<textarea>` |
| Публичное | toggle |

Создание: `POST /api/teams/[id]/events` — уведомляет участников через Telegram.

## API

`GET /api/teams/[id]/events?userId=` — список событий с `yesCount`, `status`, `venue`.

Данные загружаются при монтировании и перезагружаются после создания.
