# Таб «Финансы команды»

Сводка по деньгам команды. Видит только организатор. Считается из таблиц `FinancialTransaction` и `events`.

## Реальный баланс

```
Реальный баланс = Касса − Остаток к оплате площадкам − Переплаты игроков + Задолженность игроков
Касса = Собрано − Оплачено площадкам
```

## Показатели

| Показатель | Формула |
|---|---|
| **Собрано от игроков** | Σ `amount` из `FinancialTransaction` (все типы) |
| **Ожидаемый сбор** | Σ `price_per_player` для attendances, где `attended = true`, событие завершено |
| **Долги игроков** | по каждому: max(0, expected − paid); сумма |
| **Переплаты игроков** | по каждому: max(0, paid − expected); сумма |
| **Расходы площадкам** | Σ `events.venue_cost` |
| **Оплачено площадкам** | Σ `events.venue_paid` |
| **Остаток к оплате** | расходы − оплачено (не < 0) |
| **Касса (на руках)** | собрано − оплачено площадкам |

## Блоки страницы

1. **Hero KPI** — тёмная карточка (`bg-gray-900`):
   - Заголовок «Реальный баланс», bignum `font-display 40px` (зелёный ≥0, красный <0)
   - Три ribbon-сегмента под чертой:
     - «На счету · касса»
     - «Должны нам · игроки» (красный при `playersDebt > 0`)
     - «Должны мы · площадкам» (красный при `venueOutstanding > 0`)

2. **Сводка** — две метрики: «Собрано за всё время» + «Расходы на площадки». Остальное (долги/переплаты) уже отражено в hero.

3. **Поток за 6 месяцев** — SVG bar chart: зелёные бары (сборы) + серые (площадки) за каждый из 6 месяцев. Над каждой колонкой — компактное число (`12k`, `1.5M`). Под графиком — легенда «Сборы / Площадки» слева и «Чистый: +N ₸» справа (зелёный/красный по знаку). Данные из `GET /api/teams/[id]/insights?userId=` → `financeFlowByMonth`.

4. **Маржинальность** — горизонтальная разделённая полоса «Сборы / Расходы» (зелёный + серый). Справа от заголовка — bignum `+N ₸` или `−N ₸` (зелёный/красный). Под полосой — числа сборов и расходов. Скрыт если `collected = 0` и `venuePaid = 0`.

5. **Должны · N** — список игроков с аватаром, прогресс-баром (длина = `amount / max(amounts) × 100%`, мин. 8% для видимости) и суммой. Клик → `TeamPlayerSheet` с раскрытым аккордеоном «Финансы».

6. **Переплатили · M** — аналогично с зелёным баром. Скрыт если пусто. Клик → тот же `TeamPlayerSheet`.

7. **Площадки** — аккордеон. В шапке: «Площадки · N событий», под ней — «Оплачено N из K · долг M ₸» (долг красным). Раскрытие → список событий с `venue_cost > 0`; справа «оплачено» или «−N ₸»; клик → событие.

8. **Депозит** — inline-карточка внизу страницы: заголовок «Депозит игрока», подсказка-описание, primary-кнопка «Внести депозит». Открывает модалку: игрок + сумма + комментарий → `POST /api/teams/[id]/transactions` с `type = deposit`. Не использует `BottomActionBar` — глобальное правило, фикс-CTA не должен перекрывать нижнее меню.

## Карточка игрока

`TeamPlayerSheet` (см. `docs/features/team/[team]-roster-tab.md`) — открывается с `initialSection="finances"`, аккордеон «Финансы · в этой команде» уже раскрыт. Сальдо + история транзакций (`event_payment` / `deposit`).

## API

- `GET /api/teams/[id]/finances?userId=...` — метрики + debtors + creditors + venueEvents. Только организатор.
- `GET /api/teams/[id]/insights?userId=...` → поля `finance30d` (используется в других местах) + `financeFlowByMonth` (6 месяцев × {month, collected, venuePaid}). Только для организатора.
- `POST /api/teams/[id]/transactions` — создать транзакцию (deposit или event_payment). Только организатор.
- `GET /api/teams/[id]/transactions?userId=&player_id=&type=` — список транзакций.
- `GET /api/teams/[id]/members/[userId]/finances?userId=...` — история транзакций + сальдо конкретного игрока.
- `GET /api/teams/[id]/members/[userId]/balance?userId=...` — только баланс (expected/paid/balance + transactions).
