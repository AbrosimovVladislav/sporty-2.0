# Таб «Финансы команды»

Сводка по деньгам команды. Видит только организатор. Считается из таблиц `FinancialTransaction` и `events`.

## Реальный баланс

```
Реальный баланс = Касса − Остаток к оплате площадкам − Переплаты игроков + Задолженность игроков
Касса = Собрано − Оплачено площадкам
```

## Показатели

| Показатель | Формула |
|---|---|
| **Собрано от игроков** | Σ `amount` из `FinancialTransaction` (все типы) |
| **Ожидаемый сбор** | Σ `price_per_player` для attendances, где `attended = true`, событие завершено |
| **Долги игроков** | по каждому: max(0, expected − paid); сумма |
| **Переплаты игроков** | по каждому: max(0, paid − expected); сумма |
| **Расходы площадкам** | Σ `events.venue_cost` |
| **Оплачено площадкам** | Σ `events.venue_paid` |
| **Остаток к оплате** | расходы − оплачено (не < 0) |
| **Касса (на руках)** | собрано − оплачено площадкам |

## Блоки страницы (iter 42)

1. **Hero KPI** — тёмная градиентная карточка (`bg-gray-900`):
   - Заголовок «Реальный баланс», bignum `font-display 40px` (зелёный ≥0, красный <0) + стрелка тренда ↑/↓ (сравнение `netDelta` текущих 30д vs предыдущих 30д)
   - Строка тренда «Лучше / Хуже прошлого месяца»
   - Три сегмента под чертой: «Касса», «Долги игр.», «К оплате»

2. **Поток за 6 месяцев** — SVG bar chart: зелёные бары (сборы) + серые (площадки) за каждый из 6 месяцев. Данные из `GET /api/teams/[id]/insights?userId=` → `financeFlowByMonth`.

3. **Эффективность сборов** — `CircularProgress` 80px + `{percent}%` в центре. Рядом: «Собрано N из K ₸». Скрыт если `expected = 0`.

4. **Должны · N** — список игроков с аватаром, горизонтальным баром (красный, длина пропорциональна сумме) и суммой. Клик → карточка игрока.

5. **Переплатили · M** — аналогично с зелёным баром. Скрыт если пусто.

6. **Площадки** — список событий с `venue_cost > 0`; справа «оплачено» или «−N ₸»; клик → событие.

7. **BottomActionBar** — кнопка «Внести депозит»; модалка: игрок + сумма + комментарий → `POST /api/teams/[id]/transactions` с `type = deposit`.

## Карточка игрока

Показывает сальдо и историю **транзакций** (не attendances):
- `event_payment` — привязан к событию, отображается с датой и типом
- `deposit` — общий депозит, отображается с комментарием

## API

- `GET /api/teams/[id]/finances?userId=...` — метрики + debtors + creditors + venueEvents. Только организатор.
- `GET /api/teams/[id]/insights?userId=...` → поля `finance30d` (netDelta/prevNetDelta для тренда) + `financeFlowByMonth` (6 месяцев × {month, collected, venuePaid}). Только для организатора.
- `POST /api/teams/[id]/transactions` — создать транзакцию (deposit или event_payment). Только организатор.
- `GET /api/teams/[id]/transactions?userId=&player_id=&type=` — список транзакций.
- `GET /api/teams/[id]/members/[userId]/finances?userId=...` — история транзакций + сальдо конкретного игрока.
- `GET /api/teams/[id]/members/[userId]/balance?userId=...` — только баланс (expected/paid/balance + transactions).
