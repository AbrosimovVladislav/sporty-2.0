# Каталог команд

Публичный список команд с поиском, фильтрами и группировкой «Мои» сверху.

## Страница

- `/teams` — каталог + action «+ Создать» в зелёном хедере

## API

### GET /api/teams

Список публичных команд. Дефолтный лимит 20, максимум 100. Сортировка — `created_at desc`.

Query params:
- `q` — free-text по `name` (ilike)
- `city` — ilike
- `district_id` — точное совпадение
- `sport` — точное совпадение (на MVP только `football`)
- `looking_for_players=true` — только открытые к новым игрокам
- `exclude_ids` — csv id-шников (используется на странице, чтобы выкинуть «мои» из секции «Все остальные», когда сверху уже отрисованы они же)
- `limit`, `offset` — пагинация

Ответ:
```json
{
  "teams": [{
    "id", "name", "sport", "city",
    "description", "created_at", "looking_for_players",
    "district_id", "district": { "id", "name" },
    "members_count": 14
  }],
  "nextOffset": 20,
  "total": 42
}
```

### GET /api/teams/stats

Счётчики для green-header `/teams`. Скоупится по applied-фильтру (`city`, `sport`).

Query: `?userId=&city=&sport=`

Ответ: `{ "total": N, "mine": M | null, "lookingForPlayers": K }`

`mine` — `null`, если `userId` не передан. Иначе — число команд, в которых состоит пользователь, отфильтрованных по `city`/`sport`.

### GET /api/users/[id]/teams

Команды пользователя со всеми полями, идентичными `/api/teams` (плюс `role`). Сортировка — по `joined_at`.

Используется чтобы:
1. Показать секцию «Мои» сверху списка при дефолтной сортировке
2. Помечать строки в общей ленте бейджем «Я в составе» / «Капитан»

Поскольку у одного пользователя редко больше 5–10 команд, эндпоинт не пагинируется и фильтры применяются на клиенте.

## UI

Применение паттерна «Лэйаут листинг-страниц» (см. [docs/design/design-system.md](../../design/design-system.md)).

Структура сверху вниз:

1. **`PageHeader` (зелёный)** — title «КОМАНДЫ» + action «+ Создать» (только для авторизованного, через `HeaderActionButton`) + 3 stat-карточки: «Всего», «Мои» (только для авторизованного), «Ищут игроков». Источник — `GET /api/teams/stats`. Action ведёт на `/teams/create`
2. **`ListSearchBar`** — input «Имя команды, город…» (debounce 250ms) + filter-btn → `TeamFiltersSheet` (город, район, спорт, toggle «Ищут игроков»)
3. **`ListMeta`** — слева «Найдено N команд», справа sort-dropdown («Сначала мои» / «Сначала новые»). «Сначала мои» — дефолт; для гостя dropdown скрыт (только «Сначала новые»)
4. **`ActiveFilterChips`** — pill'ы применённых sheet-фильтров (город, спорт, «Ищут игроков»). При sheet без фильтров не рендерится
5. **Тело списка** — два режима:
   - `sort=Сначала мои` + есть мои команды → две группы под эйбрау: «Мои · N» (без пагинации, локальная фильтрация) + «Все остальные · M» (пагинация с `exclude_ids` от моих)
   - `sort=Сначала новые` (или нет моих команд) → плоский список под эйбрау «Результаты · N». В строках, где id ∈ моих, бейдж «Капитан» / «Я в составе»

`TeamListRow`: 44px Avatar (инициалы по имени команды) + название + опц. бейдж («Капитан» / «Я в составе» / «Ищут игроков», в порядке приоритета) + meta «Спорт · Район/Город» + справа `members_count` с иконкой человечка.

`TeamFiltersSheet`: bottom-sheet с чип-группами «Город» / «Район» / «Спорт» (`SheetChipGroup`) и toggle «Ищут игроков». Кнопки «Сбросить» / «Применить». Если городов 1 (на MVP — только Алматы), группа «Город» скрывается, район-чипы рендерятся сразу. Никаких нативных `<select>` в шите.

## Что осознанно не делаем

- Нет отдельного scope-чипа «Все / Только мои». Сортировка `Сначала мои` уже даёт сгруппированный вид; «Только мои» как отдельный режим не оправдан UX-ценностью на MVP. Если понадобится — добавим toggle внутри шит-фильтров
- CTA «+ Создать команду» — не отдельная карточка и не primary-кнопка снизу страницы. Это action-кнопка (pill) в правом верхнем углу зелёного хедера, рядом с местом колокольчика
- Логотип команды не реализован — `Avatar` рендерит инициал
- Сортировка «По размеру» / «По активности» — отложено
