# Каталог команд

Публичный список команд с поиском, фильтрами и быстрым переключателем «Все / Только мои».

## Страница

- `/teams` — каталог + раздел «Создать свою команду»

## API

### GET /api/teams

Список публичных команд. Дефолтный лимит 20, максимум 100.

Query params:
- `q` — free-text по `name` (ilike)
- `city` — ilike
- `district_id` — точное совпадение
- `sport` — точное совпадение (на MVP только `football`)
- `looking_for_players=true` — только открытые к новым игрокам
- `exclude_ids` — csv id-шников (используется на странице, чтобы выкинуть «мои» из секции «Все остальные», когда сверху уже отрисованы они же)
- `limit`, `offset` — пагинация

Ответ:
```json
{
  "teams": [{
    "id", "name", "sport", "city",
    "description", "created_at", "looking_for_players",
    "district_id", "district": { "id", "name" },
    "members_count": 14
  }],
  "nextOffset": 20,
  "total": 42
}
```

### GET /api/teams/stats

Счётчики для green-header `/teams`. Скоупится по applied-фильтру (`city`, `sport`).

Query: `?userId=&city=&sport=`

Ответ: `{ "total": N, "mine": M | null, "lookingForPlayers": K }`

`mine` — `null`, если `userId` не передан. Иначе — число команд, в которых состоит пользователь, отфильтрованных по `city`/`sport`.

### GET /api/users/[id]/teams

Команды пользователя со всеми полями, идентичными `/api/teams` (плюс `role`). Сортировка — по `joined_at`.

Используется чтобы:
1. Показать секцию «Мои» сверху списка при дефолтной сортировке
2. Помечать строки в общей ленте бейджем «Я в составе» / «Капитан»
3. Считать пилу `Только мои` (фильтрация на клиенте по этому списку)

Поскольку у одного пользователя редко больше 5–10 команд, эндпоинт не пагинируется и фильтры применяются на клиенте.

## UI

Применение паттерна «Лэйаут листинг-страниц» (см. [docs/design/design-system.md](../../design/design-system.md)).

Структура сверху вниз:

1. **`PageHeader` (зелёный)** — title «КОМАНДЫ» + 3 stat-карточки: «Всего», «Мои» (только для авторизованного), «Ищут игроков». Источник — `GET /api/teams/stats`
2. **`ListSearchBar`** — input «Имя команды, город…» (debounce 250ms) + filter-btn → `TeamFiltersSheet` (город, район, спорт, toggle «Ищут игроков»)
3. **`ListMeta`** — слева «Найдено N команд», справа sort-dropdown («Сначала мои» / «Недавние»). «Сначала мои» — дефолт; для гостя dropdown скрыт (только «Недавние»)
4. **`FilterPills`** (только для авторизованного) — 2 кнопки: «Все» / «Только мои». Это бинарный scope-переключатель, не комбинируемый с другими; при `Только мои` сортировка не применяется
5. **`ActiveFilterChips`** — pill'ы применённых sheet-фильтров (город, спорт, «Ищут игроков»). При sheet без фильтров не рендерится
6. **Тело списка** — три режима:
   - `scope=Все` + `sort=Сначала мои` → две группы под эйбрау: «Мои · N» (без пагинации, локальная фильтрация) + «Все остальные · M» (пагинация с `exclude_ids` от моих)
   - `scope=Все` + `sort=Недавние` → плоский список под эйбрау «Результаты · N». В строках, где id ∈ моих, бейдж «Капитан» / «Я в составе»
   - `scope=Только мои` → плоский список локально отфильтрованных «моих» (без пагинации)
7. **CTA «+ Создать свою команду»** — текстовая кнопка снизу страницы, видна только для авторизованного

`TeamListRow`: 44px Avatar (инициалы по имени команды) + название + опц. бейдж («Капитан» / «Я в составе» / «Ищут игроков», в порядке приоритета) + meta «Спорт · Район/Город» + справа `members_count` с иконкой человечка.

`TeamFiltersSheet`: bottom-sheet с город/район/спорт-select и toggle «Ищут игроков». Кнопки «Сбросить» / «Применить».

## Что осознанно не делаем

- Нет «Создать команду» как отдельной CTA-карточки или primary-кнопки в шапке. Это вторичное действие — текстовая кнопка снизу страницы
- Логотип команды не реализован — Avatar рендерит инициал
- Сортировка «По размеру» / «По активности» — отложено
