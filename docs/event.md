# Событие

Событие — любая активность команды: игра, тренировка, сбор и т.д.

## Создание события

Доступно только организатору команды.

### Экран создания
- Тип события (игра / тренировка / сбор / другое)
- Дата и время
- Площадка (выбор из существующих или создание новой)
- Стоимость участия (цена с игрока)
- Минимальное кол-во игроков для проведения
- Описание (опционально)

### Результат
Событие появляется в ленте команды. Игроки команды видят его и могут голосовать.

---

## Карточка события

Основной экран события. Видят все участники команды.

**Содержимое:**
- Тип, дата, время, площадка
- Стоимость участия
- Статус (запланировано / завершено)
- Список игроков с голосами (приду / не приду / не ответил)
- Организатор может редактировать детали

---

## Голосование

Игрок открывает карточку события и отмечает:
- **Приду** — подтверждает участие
- **Не приду** — отказывается

Голос можно изменить до начала события.

---

## Жизненный цикл события

Статусы: `planned` → `completed` / `cancelled`.

Переход выполняет **только организатор** на экране события:
- Кнопка **«Завершить»** — переводит в `completed`, открывает блоки отметок и финансов
- Кнопка **«Отменить»** — переводит в `cancelled`

Обе кнопки доступны только для событий в статусе `planned`.

---

## После события

### Игрок отмечает:
- Был на событии (да / нет)
- Сдал деньги (да / нет)

### Организатор подтверждает:
- Проверяет отметки игроков
- Может скорректировать: действительно ли был, действительно ли сдал

---

## Финансы события

По каждому завершённому событию:
- Ожидаемый сбор = стоимость × кол-во подтвердивших
- Фактический сбор = сумма от тех, кто сдал
- Разница (профицит / дефицит)

Сбора денег через приложение нет — только учёт. Деньги передаются лично, как раньше.
