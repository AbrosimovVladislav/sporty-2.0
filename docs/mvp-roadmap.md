### Итерация 11: Финансовая модель — транзакции и депозиты

#### 11.1 Модель данных

- ✅ **11.1.1** Новая таблица `FinancialTransaction`:

  | Поле         | Тип       | Описание                                                                                 |
  | ------------ | --------- | ---------------------------------------------------------------------------------------- |
  | id           | uuid      | PK                                                                                       |
  | team_id      | uuid      | FK → Team. Куда направлен платёж                                                         |
  | player_id    | uuid      | FK → User. Кто платит                                                                    |
  | amount       | numeric   | Сумма (всегда положительная)                                                             |
  | type         | enum      | `event_payment` / `deposit`                                                              |
  | event_id     | uuid?     | FK → Event (nullable, для депозитов — null)                                              |
  | note         | text?     | Комментарий (опционально)                                                                |
  | confirmed_by | uuid      | FK → User (организатор, подтвердивший). NOT NULL — транзакции создаёт только организатор |
  | created_at   | timestamp | Дата                                                                                     |

- ✅ **11.1.2** Обновить `src/types/database.ts` — тип `FinancialTransaction`
- ✅ **11.1.3** Обновить `docs/tech/db-entities.md` — новая сущность + обновлённые связи

#### 11.2 Миграция данных

## MVP 1.4++ (Отложено)

1. **Оплата площадок через транзакции** — `type = venue_payment`, команда платит площадке
2. **Трекинг результатов матчей** — победы, голы, передачи, карточки. Заполнит таб "Результаты" реальными данными
3. **Несколько видов спорта** — разные позиции и статистика по виду
4. **Персональные финансы игрока** — личная история расходов и помесячные сводки
5. Отсутствует возможность редактировать профиль команды
6. **Фото площадок** (`venues.photo_url`) — для фото-баннеров событий вместо градиент-плейсхолдеров
7. **Логотип/обложка команды** (`teams.logo_url`, `teams.cover_url`) — для team-hero на основе картинки
8. **Группировка состава по «ядру/резерву»** (как на референсе) — отдельная сущность, бизнес-логика «активный/резерв»
